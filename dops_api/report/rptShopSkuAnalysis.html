页面布局 商品修改记录
操作 响应
界面初始化 如果该界面从商品跳转过来，需要接受参数{名称，uuid，以及barcode}
此时界面商品输入框显示名称，并且根据uuid来进行搜索数据
获取商品修改记录,调用接口/{shop}/report/v2/queryShopSkuModifyLog


如果没有携带参数
时间段 默认显示 7天前的时间 （到） 今日时间
7天: 起始日期= 今天 - 6, 截至日期 = 今天
3个月: 起始日期= 今天 - 89, 截至日期 = 今天
半年: 起始日期= 今天 - 182, 截至日期 = 今天

商品输入框进行精确查询，调用接口/{shop}/shopSkuService/queryInputSearch
下拉框只展示10条数据
获取操作人列表,调用接口/{shop}/report/v2/queryAllEmployee，操作人为多选
修改内容多选下拉
下拉内容为{
"barcode":'条码',
"code":'自编码',
"lastInPrice":'最新进价',
"salePrice":'售价',
"costPrice":'成本价',
"autoRpl":'是否自动补货',
"minInvQty":'最小库存数'
}
获取商品修改记录,调用接口/{shop}/report/v2/queryShopSkuModifyLog
表格中显示，商品名称【居左】，条码【居中】，修改时间【居中】，操作人【居左】，修改内容【居左】
表格中的内容，默认显示2行，当小于等于2行时，不显示下拉箭头，若大于2行，则将2行之外的隐藏，并且显示下拉箭头
默认按照修改时间倒序来排列
商品输入回车 获取商品修改记录,调用接口/{shop}/report/v2/queryShopSkuModifyLog
点击 7天/3个月/半年 按照规则,修改 起始日期, 结束日期 输入框中的值
点击"查询" 组织查询的各项参数
获取商品修改记录,调用接口/{shop}/report/v2/queryShopSkuModifyLog
点击"重置" 查询中的各项条件, 回复初始值
点击"表格中的下拉箭头" 展开剩余数据，并且下拉箭头变成上拉箭头。
点击"表格中的上拉箭头" 收起数据，变成2行，并且下拉箭头变成上拉箭头。
点击列表中"名称/条码/修改时间/"的排序 组织查询的各项参数
获取商品修改记录,调用接口/{shop}/report/v2/queryShopSkuModifyLog
