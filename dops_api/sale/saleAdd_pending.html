挂单初始化 该界面入口为销售界面上的"挂单/恢复挂单"
如果当前销售商品行数量 == 0, 是"恢复挂单", 否则是挂单

if (localStorage.getItem('{shop}.sale-pendingSaleOrder') == null), 则创建此对象, 并插入9条空记录
"挂单", 点击一个空的窗格 将当前单据的数据对象完整存入, 关闭挂单界面
商品录入的信息全部清空
"挂单", 点击一个已有单据的窗格 提示用户"是否替换"?
否: 此操作取消
是: 将当前单据的数据对象完整存入替换, 关闭挂单界面
　　商品录入的信息全部清空
"恢复挂单", 点击一个空的窗格 无效
"恢复挂单", 点击一个已有单据的窗格 将当前单据的数据对象完整存入销售单的对象, 关闭挂单界面
